System Pipeline

01
Data Preprocessing

Raw SMS text is cleaned — lowercased, stripped of noise, and stopwords removed — to produce a normalised input.

↓
02
TF-IDF Feature Extraction

Term Frequency–Inverse Document Frequency converts processed text into high-dimensional numerical feature vectors.

↓
03
SVM Classification

A trained Support Vector Machine finds the optimal hyperplane separating spam/smishing from legitimate messages.

↓
04
Real-time Prediction

Results are returned via a REST API endpoint and rendered instantly in the interface.

Why SVM?

High-Dimensional Performance

SVM excels in sparse, high-dimensional feature spaces — exactly the environment TF-IDF produces from text data.

Strong Generalisation

Maximum-margin classification reduces overfitting, meaning the model generalises well to unseen SMS messages.

Efficiency

Lightweight inference means sub-50 ms predictions, supporting real-time applications without heavy infrastructure.

Proven on Text Tasks

SVM has a long track record in NLP benchmarks, particularly for spam and sentiment classification tasks.
