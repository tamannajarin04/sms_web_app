Technical Architecture

ML Model

The algorithm, vectorisation strategy, and inference pipeline behind SmishGuard.

⬡ Algorithm Support Vector Machine
◈ Vectorisation TF-IDF
◉ Task Binary Classification
◫ Output Spam · Legitimate

Inference Pipeline

01
Data Preprocessing

Raw SMS is lowercased, stripped of punctuation and noise, and stopwords are removed to produce a clean token stream.

02
TF-IDF Feature Extraction

Tokens are transformed into a high-dimensional sparse numerical vector using a fitted TF-IDF vectoriser.

03
SVM Classification

The feature vector is passed to a trained linear SVM which finds the maximum-margin hyperplane separating spam from ham.

04
Real-time Prediction

The label and confidence are serialised as JSON and returned by the REST endpoint in under 50 ms.

Why SVM?

High-Dimensional Performance

SVMs excel in sparse, high-dimensional spaces — exactly what TF-IDF produces from text corpora.

Strong Generalisation

Maximum-margin classification minimises overfitting, so the model holds up on unseen messages.

Lightweight Inference

No GPU required. A serialised SVM predicts in microseconds, enabling sub-50 ms API responses.

Proven on NLP Tasks

SVMs have a strong benchmark track record on spam, sentiment, and short-text classification.
